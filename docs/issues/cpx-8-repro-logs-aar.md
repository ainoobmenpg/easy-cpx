# CPX-8: 再現性（RNGシード）＋構造化ログ＋AAR拡充

### 背景
審判は乱数を用いるが、完全再現の仕組みと審判根拠のログが限定的。AARは拡張余地が大きい。

### 提案
- RNG: ターン/判定単位のシード固定と永続化。CLI/環境変数で再実行可能に。
- ログ: `adjudication_log`/`inject_log` をJSONで保存（前提チェック結果、ロール値、修正内訳）。
- AAR: MOP/MOE/達成度指標・星評価・改善提案を自動出力。

### 受け入れ基準
- 同一シード・同一入力で判定が再現できる（テストで保証）。
- AARに前提/判定/結果/教訓の因果が追える形で出力される。
